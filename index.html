Soco Magicalladin

Tech Stack

Laravel

About this project

Soco Magicalladin is a social commerce platform with four user roles: Seller, Supplier, Admin, and Kurir (IDL). Sellers can sell without capital by registering, browsing products from suppliers, and sharing links for buyers to checkout. Suppliers upload and manage the product catalog, purchase funds are financed (advanced) to suppliers with a fee. Admin manages the platform; courier (IDL) handles delivery operations.

Features

  • Seller — Register, browse products, share product links for buyers; sell without capital (screens: soco2–soco5)
  • Supplier — Upload and manage products for sellers to resell (screen: soco6)
  • Admin — Platform management, monitoring, and configuration (screens: soco7, soco8)
  • Kurir (IDL) — Courier/delivery interface for order fulfillment (screen: soco9)
  • Payment gateway Duitku
  • Integration with marketplace APIs (Shopee, Tokopedia, Lazada)
  • Financing (advance) to suppliers with fee-based model

Customs BC

Tech Stack

Laravel Queue Job

About this project

Customs BC is a web app for PLB ecommerce that helps calculate warehouse rental costs, including VAT, income tax, import duty, and CIF. Previously these calculations were done in Excel and took a long time, this app significantly shortens processing time. It handles tens of thousands of inbound and outbound items from various clients, one of the largest is Lazada. Users work from a provided Excel template, import data into the app, and processing runs via Laravel queue jobs to handle the large volume.

Features

  • Cost calculation: PPH, PPh, bea masuk, CIF for warehouse rental
  • Import data using provided Excel template
  • Laravel queue job for processing large data volumes
  • Inbound and outbound management (tens of thousands of records)
  • Multi-client support (e.g. Lazada)
  • Replaces manual Excel-based calculation to save time

Finance

Tech Stack

Laravel Cron job

About this project

Finance is a web app for managing cash flow from request to disbursement and audit. Employees submit cash advance or settlement, the request then goes through HOD (Head of Director) approval, Finance approval, and Cashier approval. Once the cashier approves, funds are disbursed for the employee’s use. The flow continues to Accounting for audit and record-keeping.

Features

  • Employee submission: ash advance and settlement
  • Multi-level approval: HOD → Finance → Cashier
  • HOD (Head of Director) approval
  • Finance department approval
  • Cashier approval and disbursement (uang cair)
  • Accounting module for audit and records
  • Request, approval, and reporting workflow
  • Reminder approval via cron job

RPA Manifest PLB Ecommerce

Tech Stack

UIBot Python

About this project

RPA Manifest PLB Ecommerce is a robotic process automation (RPA) solution that automates the gate-out process in PLB ecommerce operations. Previously, this process was done manually by employees through repeated clicking, the RPA reduces the need to hire staff for this repetitive work and lowers operational cost while keeping the flow reliable—including manifest submission to Bea Cukai (Customs).

Features

  • Automation of gate-out process in PLB ecommerce
  • Replaces manual, repetitive clicking by employees
  • Cost reduction by reducing headcount for repetitive tasks
  • Manifest handling and submission to Bea Cukai (Customs)
  • Built with UIBot and Python
